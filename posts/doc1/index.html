Créer un dépôt GitHub avec GitHub CLI et lier un projet sur MacOS

Prérequis

  1. Avoir un compte GitHub.
  2. Installer GitHub CLI (gh). 
    brew install gh
  3. Installer Git si ce n’est pas déjà fait. 
    brew install git

1. Se connecter à GitHub via GitHub CLI

Pour utiliser la CLI GitHub, il est nécessaire de se connecter à ton compte GitHub via le terminal.

gh auth login

Suis les instructions pour te connecter à ton compte GitHub.

2. Créer un dépôt GitHub depuis le terminal

Utilise la commande suivante pour créer un dépôt sur GitHub :

gh repo create NOM_DU_REPO --public
  • Remplace NOM_DU_REPO par le nom que tu souhaites donner à ton dépôt.
  • L’option --public crée un dépôt public, mais tu peux utiliser --private pour un dépôt privé.

Tu seras invité à choisir si tu veux créer un dépôt local ou distant. Sélectionne “Push an existing local repository”.

3. Créer un projet local

Si tu n’as pas encore de projet local, crée-en un en initialisant un nouveau répertoire avec Git.

mkdir mon-projet
cd mon-projet
git init

Cela initialise un nouveau dépôt Git local dans le répertoire mon-projet.

4. Lier le projet local au dépôt GitHub

Maintenant que le dépôt GitHub est créé, tu dois le lier à ton dépôt local.

a. Ajouter le dépôt GitHub en tant que remote

git remote add origin https://github.com/ton-utilisateur/NOM_DU_REPO.git

Remplace ton-utilisateur par ton nom d’utilisateur GitHub et NOM_DU_REPO par le nom du dépôt que tu as créé.

b. Ajouter un fichier au projet local

Ajoute un fichier pour le test, par exemple un fichier README.md :

echo "# Mon projet" > README.md

c. Ajouter, valider et pousser les changements vers GitHub

  1. Ajouter les fichiers dans Git :

    git add .

  2. Valider les fichiers ajoutés :

    git commit -m "Premier commit"

  3. Pousser les changements vers le dépôt GitHub :

    git push origin master

5. Vérifier sur GitHub

Ton projet local est maintenant lié à ton dépôt GitHub. Tu peux vérifier en te rendant sur GitHub et en consultant ton dépôt.
